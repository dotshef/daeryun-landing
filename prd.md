# PRD — 대륜 1급 공업사 랜딩 페이지

## 1. 프로젝트 개요

- **프로젝트명**: daeryun-landing
- **목적**: "사고 수리비 0원 이벤트"를 내세워 자차보험 가입자의 수리 상담 신청(리드)을 확보하는 전환형 랜딩 페이지 개발
- **레퍼런스**: https://repair-factory.vercel.app/ — 구조·흐름·인터랙션을 동일하게 구현
- **핵심 차이점**: 레퍼런스의 콘텐츠 이미지를 자체 제작한 "대륜" 브랜드 이미지(`reference/01_section.jpg` ~ `11_section.jpg`)로 교체
- **타겟 사용자**: 서울/경기/인천 거주, 자차보험 가입, 차량 파손(긁힘/찌그러짐/파손) 상태의 차주. 대부분 모바일(광고 유입)로 접속

## 2. 목표 (성공 기준)

| 목표 | 지표 |
|---|---|
| 리드 확보 | 신청 폼 제출 수 (이름 + 연락처) |
| 즉시 상담 연결 | 전화상담 / 카톡상담 버튼 클릭 수 |
| 이탈 최소화 | 이탈 방지 팝업, 반복 CTA 배치로 스크롤 완주율 확보 |
| 빠른 로딩 | 모바일 LCP 2.5초 이내 (첫 이미지 최적화) |

## 3. 기술 스택 (현재 프로젝트 그대로 사용)

- **프레임워크**: Next.js 16 (App Router) + React 19 + TypeScript
- **스타일**: Tailwind CSS 4
- **이미지**: `next/image` (jpg → webp 변환 후 `public/images/`에 배치)
- **배포**: Vercel
- **추가 라이브러리 최소화**: 애니메이션·아코디언·모달 모두 CSS/React 상태만으로 구현 (외부 UI 라이브러리 불필요)

## 4. 페이지 구조

레퍼런스와 동일하게 **단일 페이지(`/`) 세로 스크롤** 구성. 콘텐츠는 모바일 폭 중심(최대 너비 480px 컨테이너, 데스크톱에서는 중앙 정렬)으로 렌더링한다.

배치 순서 (레퍼런스 HTML 실측 — 그대로 복제):

| 순서 | 요소 | 콘텐츠 |
|---|---|---|

| 13 | **신뢰 배지 바** | 4개 배지: "1급 공업사 인증 / 20년+ 수리 경력 / 100% 고객 만족 / 0원 수리비" |

- **이미지 01~10은 사이에 아무 요소 없이 연속 배치**된다. 폼은 페이지 최하단에 1개만 존재하고, 대신 하단 고정 퀵 접수 바가 스크롤 내내 따라다닌다.
- 이미지 04, 11은 클릭 시 최하단 신청 폼으로 스크롤되는 앵커 링크다.

## 5. 기능 요구사항

### 5.1 실시간 접수 티커
- 페이지 최상단에서 접수 내역이 일정 간격(3~4초)으로 교체/롤링되는 애니메이션
- 데이터는 하드코딩된 목업 배열(이름 마스킹 "김○○" + 차종 + "n분 전")을 순환 표시
- CSS transition 또는 간단한 setInterval 기반 구현

### 5.2 신청 폼 (본문 최하단, `#apply-form`)
- 타이틀: "무료 수리 신청하기" / 서브: "30초만에 간편 접수, 빠른 상담 연결!"
- 필드:
  - 이름 (필수, placeholder "홍길동")
  - 연락처 (필수, placeholder "010-0000-0000" — 자동 하이픈 포맷, 010 형식 검증)
  - 차종 (선택, placeholder "예: 아반떼, BMW 320 (모르시면 비워두세요)")
  - 개인정보 수집·이용 동의 (필수 체크박스 — 아래 동의 문구 명시)
- **개인정보 수집·이용 동의 안내 문구** (폼 내 노출):
  - 수집항목: 이름, 연락처, 차종
  - 이용목적: 수리 상담 및 견적 안내
  - 보유기간: 상담 종료 후 3개월
  - 수집·이용에 거부할 수 있으며, 거부 시 상담이 불가합니다.
- 제출 버튼: **"지금 바로 무료 접수하기"** / 버튼 하단 안내: "접수 후 5분 이내 전문 상담사가 연락드립니다."
- 유효성 미충족 시 제출 차단 + 필드별 에러 안내
- 제출 성공 시 **접수 완료 모달** 표시
- **제출 처리**: Next.js Route Handler(`/api/apply`)로 POST → 수신 채널로 전달 (오픈 이슈 참조)

### 5.3 FAQ 아코디언 — "자주 묻는 질문" (레퍼런스 원문 그대로, 브랜드명만 "대륜"으로 교체)

| # | 질문 | 답변 |
|---|---|---|
| 1 | 정말 수리비가 0원인가요? | 네, 자차보험 가입자 중 서울/경기/인천 거주자, 파손된 차량 보유자에 해당되시면 자기부담금 포함 수리비 전액을 지원해드립니다. |
| 2 | 자차보험 쓰면 내년에 할증 붙지 않나요? | '물적사고 할증기준 금액' 안에서 수리하면 보험료 할증이 발생하지 않습니다. 대륜은 할증기준 안에서 완벽 복원 가능하도록 견적을 설계해드립니다. 상담 시 고객님 가입 보험의 할증기준을 함께 확인해드립니다. |
| 3 | 어떤 차종이든 가능한가요? | 국산차, 수입차 모두 가능합니다. 차종과 파손 정도에 따라 상담 후 안내드립니다. |
| 4 | 탁송은 어떻게 진행되나요? | 서울, 경기, 인천 지역은 편도 무료 탁송을 지원합니다. 전화 한 통이면 차량 픽업부터 수리 완료 후 배송까지 모두 처리해드립니다. |
| 5 | 수리 기간은 얼마나 걸리나요? | 파손 정도에 따라 다르지만, 일반적으로 3~5일 이내에 완료됩니다. 꼼꼼한 검수 후 출고해드립니다. |
| 6 | 이 이벤트는 언제까지인가요? | 1급 공업사 확장 기념 이벤트로, 선착순 마감입니다. 조기 마감될 수 있으니 빠른 접수를 권장드립니다. |

- 질문 클릭 시 펼침/접힘 토글 (열림 시 `+` → `−` 아이콘 전환), 한 번에 여러 개 열림 허용

### 5.4 하단 고정 퀵 접수 바 (bottom-fixed)
- 화면 하단에 fixed, 최대 너비 480px 중앙 정렬, 다크 배경
- 구성 (2줄):
  1. `[필수] 개인정보 수집·이용 동의 (자세히)` 체크박스 — "(자세히)" 클릭 시 본문 폼(`#apply-form`)으로 이동
  2. 이름 입력 + 연락처 입력 (인라인) + **"무료 접수하기"** 버튼
- 제출 성공 시 본문 폼과 동일하게 접수 완료 모달 표시
- 본문 최하단에 퀵 바 높이만큼 spacer를 두어 콘텐츠 가림 방지

### 5.5 플로팅 상담 버튼 (우측 하단 원형 2개)
- **전화 버튼**: 초록색 원형(64px), `tel:{전화번호}`, 펄스 애니메이션 + "전화 상담" 말풍선 라벨
- **카톡 버튼**: 카카오 노란색 원형(64px), 카카오톡 채널 채팅 URL 새 창, 펄스 애니메이션 + 빨간 알림 뱃지
- 퀵 접수 바 위쪽에 세로로 나란히 배치 (bottom 간격 차등)

### 5.6 모달 2종
- **접수 완료 모달**: 폼 제출 성공 시 "접수가 완료되었습니다. 곧 연락드리겠습니다" + 닫기
- **이탈 방지 모달**: 이탈 시도 감지 시 "지금 나가시면 이 이벤트 신청이 불가합니다" 경고 후 폼으로 유도. 사용자가 페이지와 상호작용한 이후 + 세션당 1회만 표시 (레퍼런스 동일). 모바일 특성상 `beforeunload`보다 뒤로가기(popstate) 감지 방식 사용

### 5.7 신뢰 배지 바
- 이미지 11 바로 아래, 신청 폼 직전 배치
- 4개 배지: **1급** 공업사 인증 / **20년+** 수리 경력 / **100%** 고객 만족 / **0원** 수리비

## 6. 이미지 처리

- `reference/*.jpg` 11장을 webp로 변환(sharp 스크립트 활용)하여 `public/images/section-01.webp` ~ `section-11.webp`로 배치
- 모든 섹션 이미지는 `next/image`로 렌더 (원본 비율 유지, `sizes` 지정)
- 이미지 01(히어로)만 `priority` 지정 → LCP 최적화, 나머지는 lazy 로딩
- `alt`에 각 섹션의 핵심 문구를 넣어 접근성·SEO 확보 (이미지 기반 페이지의 텍스트 부재 보완)

## 7. SEO / 메타데이터

- `<title>`: "대륜 1급 공업사 | 사고 수리비 0원 이벤트"
- description, Open Graph(og:image = 히어로 이미지), viewport 설정
- 이미지 중심 페이지이므로 주요 카피를 `alt` + 스크린리더용 텍스트로 보강

## 8. 비기능 요구사항

- **반응형**: 모바일 우선. 데스크톱에서는 콘텐츠 컬럼 중앙 정렬(배경색 처리)
- **성능**: 이미지 lazy 로딩, webp 변환, 폰트는 시스템 폰트 또는 Pretendard 1종만
- **접근성**: 폼 라벨링, 모달 포커스 처리, 이미지 alt
- **분석**: (선택) GA4 또는 Vercel Analytics로 CTA 클릭·폼 제출 이벤트 추적

## 10. 구현 단계

1. **이미지 준비**: jpg → webp 변환, `public/images/` 배치
2. **정적 레이아웃**: 티커 + 섹션 이미지 11장 + 신뢰 배지 바로 페이지 골격 완성 (레퍼런스 순서 그대로)
3. **인터랙션**: 신청 폼 + 퀵 접수 바(검증 포함), FAQ 아코디언, 플로팅 버튼, 모달 2종
4. **폼 백엔드**: `/api/apply` 구현 + 수신 채널 연동
5. **마무리**: SEO 메타, 성능 점검(Lighthouse), 반응형 확인, Vercel 배포
